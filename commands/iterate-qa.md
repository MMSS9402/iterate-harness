당신(메인 Claude)이 **배치 QA 드라이버**다. `${CLAUDE_PLUGIN_ROOT}/skills/iterate-protocol/SKILL.md`(SSOT)의 격리 규칙 아래, `/iterate`(lean)가 미뤄 둔 explorer 탐색을 **한 번의 환경 기동으로 묶어** 수행한다. 목적: 카드마다 실물 기동+탐색을 반복하던 비용을 상각하고, **인접 카드 표면의 상호작용**(단일 카드 탐색이 원리상 못 보는 것)까지 한 번에 본다.

**인자**: $ARGUMENTS

## 0) 준비
- SSOT 와 `.claude/iterate.config.md` 어댑터를 Read. `EXPLORE_QA`·`ARTIFACTS_DIR` 필수(없으면 중단·어댑터 보강 요청).
- `$ARTIFACTS_DIR/qa_queue.md` 를 Read — 인자에 카드 ID 가 있으면 그 카드만, 없으면 큐 전체가 대상. 큐가 비어 있고 인자도 없으면 "QA 대기 카드 없음" 보고 후 종료.
- 대상 카드들의 등록 블록(요약·AC 1줄·변경 파일·탐색 초점·시드/기동 힌트)을 취합한다. 카드가 많으면(>5) 사용자에게 분할 여부 1줄 확인.
- `mkdir -p "$ARTIFACTS_DIR/explore/batch-<날짜>"`.
- **가드 pre(당신 직접)**: `{ git status --porcelain .; git diff HEAD -- .; git ls-files -o --exclude-standard -z . | xargs -0 shasum; } | shasum` 기록(탐색자 무수정 백스톱 — /iterate §4b 와 동일).

## 1) 환경 기동 (1회)
어댑터 `EXPLORE_QA` 기동법으로 실행 환경(서버/시뮬레이터)을 **한 번** 띄운다(백그라운드). 기동 불가면 **탐색 생략 종결 금지** — 중단하고 사용자에게 기동 요청(SSOT 원칙).

## 2) 탐색 fan-out (Workflow 허용 — SSOT §에이전트 독립성 예외)
- **작업 목록 구성**: 대상 카드×탐색 초점(카드당 기본 5개 상한 — 인자로 조절)을 항목화하되, **여러 카드가 공유하는 표면은 하나의 교차 항목으로 병합**한다(같은 화면을 카드별로 중복 탐색하는 낭비 차단). 추가로 **카드 간 상호작용 항목**(A 카드 조작 후 B 카드 표면 상태)을 1~2개 넣는다 — 배치의 고유 가치.
- **실행**: 항목 수가 3 이하면 순차 `Agent(iterate-harness:explorer)` 호출로 충분하다. 4 이상이면 `Workflow` 로 병렬 fan-out — 각 fan-out 에이전트 프롬프트에는 해당 항목의 카드 요약·초점·기동된 환경 접속 정보·**격리 규칙**(소스 루트·test/ 무수정, Write 는 `$ARTIFACTS_DIR/explore/batch-<날짜>/` 만, 발견은 재현 절차+기대 vs 실제+증거 경로로 반환)을 명시한다. 탐색 판정·결함 집계는 Workflow 로 이동하지 않는다 — 당신이 반환값을 직접 집계한다.
- 각 항목은 시간상자를 지킨다(항목당 흐름 5~8개 — SSOT explorer 규정).

## 3) 집계·기록
- **가드 post**: pre 와 같은 커맨드 재비교 — 달라졌으면 탐색자가 제품/테스트를 수정한 것 → 원복 지시 후 재확인.
- 발견 결함을 `$ARTIFACTS_DIR/qa_findings.md` 에 카드별로 append: `## [<카드 ID> · batch-<날짜>]` 헤더 + 각 결함(심각도·재현 절차·기대 vs 실제·증거 경로·테스트 환원 방안 제안). 카드에 못 귀속되는 교차 결함은 `## [교차 · batch-<날짜>]` 로.
- 환경 종료(기동한 프로세스 정리).

## 4) 라우팅·종결
- **결함 0 인 카드** → `PLAN_PATH`(설정 시) 완료기록의 `green (QA 대기)` 를 **done** 으로 갱신(QA 배치·날짜 명기). `qa_queue.md` 에서 해당 블록 제거.
- **결함 ≥1 인 카드** → 큐에서 제거하되 완료기록에 `QA 결함 N건 — /iterate <카드> 재실행 필요` 를 남기고, 사용자에게 카드별 결함 요약과 함께 **`/iterate <카드>` 재실행**을 안내한다(재실행 사이클이 failures 경로로 test-author 환원→G2→재동결→게이트를 태운다 — 환원을 이 배치가 직접 하지 않는 이유: 게이트 판정·동결 권한은 카드 사이클의 드라이버 직렬 책임, SSOT). 교차 결함은 가장 관련 깊은 카드에 귀속시키거나 신규 카드 제안.
- **종료 보고**: 대상 카드 수·탐색 항목 수(병합·교차 포함)·결함 카드/건수·done 갱신 목록·재실행 필요 목록·총 소요.
